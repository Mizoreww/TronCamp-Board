01 / 概览

力觉,是视觉看不见的那一半世界

robosuite 的 Wipe 是天然的力觉密集任务:机械臂以受控接触力沿桌面擦除污渍—— 压力不足擦不掉、压力过大触发超力惩罚。机械爪腕部内置六维力/力矩传感器, 整场比赛的力信号即由此而来。

全卷统一用 π0.5(~3B,FluxVLA 旗舰)、单任务 Wipe、单仿真栈,比的是力觉算法而非模型大小。 开放挑战:主办方下发专家轨迹 bag / 数据集与评测;数据由主办方给定、不可重新采集,数据处理 / 模型 / 训练自己来。

唯一评测环境:每 episode 桌高在 ±3cm 内随机偏移。提交一个模型、评测一次,按成功率与力质量导出 T1 / T2 / T3 三档,顺序解锁(过 T1 → 过 T2 → 进 T3 主榜)。偏移相机难以直读, 纯视觉策略会戳桌 / 悬空,会用力的策略才能稳擦——擦除率由此拉开。

力觉的价值 · 三段评测 rollout

同一桌高偏移评测 seed(本例 −2.9cm)下的真实回放;右侧实时显示接触法向力 |Fz| 与专家带 P10–P90(3.0–10.3 N)。

02 / 赛题

三道递进必做题

每队提交一个模型(你的代码 + 权重),在唯一评测环境(桌高 ±3cm 随机偏移、固定 seed 表全场一致)用你的代码评测一次;T1 / T2 / T3 由这一条成绩导出、顺序解锁。三档都跑你的代码,差别只在判定口径:T1 / T2 达标即满分,主榜只排 T3 绝对分。

档名称内容计分
T1管线通关用给定的专家轨迹 bag / 数据集自建数据处理与训练管线,跑通"数据 → 训练 → 上榜";处理与 I/O 完全放开(数据不可重新采集)。成功率 SR ≥ 0.25 → 达标满分
T2力觉接入接入六维力/力矩,在 ±3cm 偏移下稳擦;可用脚手架的力觉接入示例。SR ≥ 0.55 → 达标满分(须先过 T1)
T3力觉改进
全卷重心 · 主榜		任意改进力觉算法(数据 / 表征 / 力注入 / 架构),在 ±3cm 下冲最高分;C / D 级改进在你的代码里生效。T3 = 0.7·擦除率 + 0.3·力质量分,降序排名(须先过 T2)

T3 改进方向(仅供参考)

以下只是一些可尝试的方向,非强制、不分级、不单独计分 —— 主榜只看 T3 绝对分。数据由主办方给定、不可重新采集,改进只在给定数据与你的代码上做。

类别可尝试方向
A 数据级在给定数据上做力觉数据增广 / 过滤 / 重加权(不可重新采集)
B 表征级B1 滑动窗力历史 · B2 频域 / 统计特征 · B3 压缩归一化方式 · B4 离散 bins 分辨率
C 注入通道级C1 连续 force token 注入 action expert · C2 离散 + 连续双通道 · C3 注入位点 prefix vs suffix
D 架构 / 策略D1 辅助力预测头 · D2 接触相位门控 · D3 力蒸馏(推理免传感器) · D4 小 MoE 融合 · D5 视觉退化课程 · D6 SARM 阶段加权
03 / 比赛规则

赛制与红线

赛制10 天纯线上,每队 2×A100-80G。开放挑战:主办方下发专家轨迹 bag + 评测 harness + 可选脚手架(含力觉接入起步示例)。数据由主办方给定、不可重新采集;在给定数据上的处理 / 模型 / 训练全自由。单模型 π0.5、单任务 Wipe、单仿真栈,版本锁定 FluxVLA 指定 commit,赛中不跟上游。
评测条件唯一环境:桌高 ±3cm 随机偏移,固定 seed 表全场一致。每模型 dev 50 / final 100 个独立 seed episode 跑一次。dev seed 表(含 corrupter salt)随脚手架公开——本地评测分 = dev 榜分,可本地自测。
提交每队每日(UTC+8)≤ 2 次,超额拒绝。截止时刻提交通道关闭,当时的 dev 榜即最终成绩(榜单页有倒计时)。
红线① 单次提交、用你上传的代码评测一次,T1 / T2 / T3 同出这一条成绩;数据处理 / IO / 力注入 / 模型 / 训练方法完全放开,主办方不审查代码、只看统一评测的达标与分数;② 自定义模块 / 表征 / 注入在你的代码里全生效(T3 的 C / D 级改进据此计分);③ 推理期只用白名单观测,仿真内部状态 / 私有 seed 碰不到(ZMQ 进程隔离 + 沙箱);④ Final 名次用私有 seed 表统跑,防 dev 过拟合;⑤ 伪造评测或绕过隔离,单题零分起。
04 / 评分标准

不只看成功率

单看成功率测不出"力"的价值:超压硬怼也能擦完,SR 好看而力行为糟。故主榜 T3 用连续的擦除率与力质量分,而非二值成功率。

主榜 = T3 绝对分 = 0.7·擦除率 + 0.3·力质量分 （只排 T3;分数越高名次越前）

逐档计算口径

三档全部由同一次评测(每模型 dev 50 / final 100 个独立 ±3cm seed episode)导出,顺序解锁。

档指标计算方式判定 / 排名
T1成功率 SRSR = 成功 episode 占比;单 episode 成功 = 擦除率 ≥ 90%。SR ≥ 0.25(低阈值,±3cm 实测标定)→ 达标满分(绿勾)
T2成功率 SR同一条成绩的 SR,不另跑评测。SR ≥ 0.55(高阈值)→ 达标满分;须先过 T1(±3cm 下基本需用力才过)
T3绝对分0.7·擦除率 + 0.3·力质量分;擦除率 = 各 episode 擦除百分比的均值(连续量,非二值)。须先过 T2 解锁;按 T3 绝对分降序排名(主榜)
力质量分接触力行为
(进 T3,权重 0.3)		0.4·带内 + 0.2·(1−超压) + 0.2·(1−峰值惩罚) + 0.2·(1−抖动惩罚),均在接触相、对真实法向力 |Fz| 计,由 scorer 从落盘力轨迹重算。带内 = |Fz|∈[3.0, 10.3] N 占比;超压 = |Fz| > 1.5·P90 占比;峰值惩罚 = 峰值超 2·P90 程度;抖动 = 相邻帧力差标准差
05 / 提交方式

上传代码包 + 权重,自动评测上榜

不交公开 fork(会泄露解法)——上传你的代码压缩包 + 权重。评测机解压后用你的代码起推理服务,在 ZMQ 进程隔离 + 沙箱里跑分;成功率与力质量分均由主办方 harness 从仿真算出,选手自报无效。每队每日(UTC+8)≤ 2 次。

网页提交 ↗ 浏览器选文件直传权重 / 代码包,后端自动算 SHA256,无需装环境。

流程:① 用给定数据(专家轨迹 bag / 数据集)自建处理与训练,不可重新采集(可用脚手架);② 改过的代码(自定义 I/O、力注入、模型结构)打包 .zip / .tar.gz,权重单独导出;③ 网页或 CLI 上传代码包 + 权重(两者留空则跑官方力觉基线保底);④ 评测机隔离起服务、score_force.py 跑分 → Dev 榜实时、Final 截止冻结。

也可命令行 / 自动化提交(大文件、批量更稳):

# 上传代码包 + 权重,自动算 SHA256 python scripts/submit.py --server http://<评测服务器>:8080 --token <队伍令牌> \ --code-file ./mycode.zip --ckpt-file ./force.pt

你可自由改数据处理 / 表征 / 力注入 / 架构(T3 改进方向 A–D);但只拿得到 ZMQ 白名单观测、读不到私有 seed,分数由 harness 算 —— 改 I/O 是开放挑战要的,作弊作不了。

最终截止前还需提交

赛中只需上传代码包 + 权重自动评测上榜;最终截止前另需提交以下两项,用于复现核验与评审:

开源代码 + 设计文档将完整代码开源到 GitHub,并附技术设计文档——详细写明环境配置、运行指令、算法逻辑(须可复现)。
演示视频时长严格 ≤ 3 分钟,以屏幕录制 + 画中画(摄像头)形式:本人出镜讲解算法思路,并现场 Demo 运行效果。

详见 学生手册 与 T3 改进方向对照表。
