01 / 概览

力觉,是视觉看不见的那一半世界

robosuite 的 Wipe 是天然的力觉密集任务:机械臂要以受控接触力沿桌面 擦除污渍——压力不足擦不掉、压力过大触发超力惩罚。机械爪腕部内置六维力/力矩 传感器,整场比赛的力信号即由此而来。

全卷统一用 π0.5(~3B,FluxVLA 旗舰)、单任务 Wipe、单仿真栈—— 榜单上比的是力觉算法,不是谁的模型大。开放挑战:主办方只给专家轨迹 bag 与 评测,数据处理/模型/训练全自己来。T1/T2 是达标门(达标即满分), 主榜只排 T3 绝对分,决定名次。

全卷只有一个评测环境:"桌高 ±3cm 随机偏移 + 视觉遮挡"。T1/T2/T3 三档由同一条成绩 按阈值/绝对分导出——偏移相机难以直读,纯视觉策略会戳桌/悬空,会用力的策略才能稳擦, 擦除率由此拉开。

02 / 赛题

三道递进必做题

单一评测环境:桌高 ±3cm 偏移 + 视觉遮挡(固定 seed 表全场一致),每队提交一个模型只评测一次,T1/T2/T3 三档由同一条成绩导出。所有量化指标由评测集群自动计算,逐 episode 配对。T1/T2 达标即满分,主榜只排 T3 绝对分。

T1达标

管线通关

入门 · ±3cm 低阈值门 · 达标即满分
  • 从专家轨迹 bag 起,自己搭数据处理与训练,跑通"数据→训练→上榜"
  • 同一 ±3cm 扰动评测下成功率 ≥ 主办方实测标定的低阈值即满分(绿勾)
  • 怎么实现都行——数据管线与 I/O 完全放开
T2达标

力觉接入

进阶 · ±3cm 扰动 · 达标即满分
  • 接入六维力,在桌高 ±3cm 偏移 + 视觉遮挡下稳擦
  • 同一 ±3cm 扰动评测下成功率 ≥ 高阈值即满分(绿勾)
  • 同样放开数据管线 + I/O;参考脚手架给现成力觉接入示例
T3主榜

力觉改进竞技场

全卷重心 · 决定名次
  • 任意改进力觉算法,在 ±3cm 扰动下冲最高分
  • 用你 fork 的代码评测:自定义模块/表征/注入全生效
  • 主榜 = T3 绝对分 0.7·擦除率 + 0.3·力质量分,按分排名

T3 改进点菜单 · 15 项 · 四级

量化效果上榜不分级;新颖性分按级封顶。A/B 级 0 行模型代码、人人能上场;C/D 级决定头部名次。

03 / 比赛规则

赛制与红线

赛制

10 天纯线上;每队 2×A100-80G。开放挑战:主办方只下发专家轨迹 bag + 评测 harness + 可选参考脚手架(含力觉接入与 ForceVLA 连续力 token 示例),数据处理/模型/训练全自由。全卷单模型 π0.5、单任务 Wipe、单仿真栈,版本锁定 FluxVLA 指定 commit,赛中不跟上游。

评测条件

单一评测环境:桌高 ±3cm 随机偏移 + 视觉遮挡。每队模型 dev 50 / final 100 个独立场景 seed episodes 跑一次,逐 episode 配对;T1/T2/T3 三档由这一条成绩按阈值/绝对分导出。dev seed 表(含 corrupter salt)随脚手架公开——本地评测分 = dev 榜分,选手可本地自测。

提交限额

每队每日(UTC+8)至多 2 次提交,超额拒绝。截止时刻提交通道关闭,当时的 dev 榜即最终成绩(榜单页有倒计时)。

红线(违反量化分作废)

① T1/T2 数据处理与 I/O 完全放开,不查代码,只看官方评测达标(成功率 ≥ 主办方实测标定阈值);② 模型/力注入/训练策略是 T3 的战场,T3 用你 fork 的代码起服务、自定义模块全生效;③ 推理期只用白名单观测,仿真内部状态/私有 seed 推理期碰不到(ZMQ 进程隔离 + 沙箱);④ Final 名次用私有 final seed 表统跑,防 dev 过拟合;⑤ 负结果如实报告照样计分,伪造评测或绕过隔离单题零分起。

04 / 评分标准

不只看成功率

SR 是基础层,但单看 SR 测不出"力"的价值——一个"成功但暴力"的策略(超压硬怼也能擦完)SR 很好看、力行为很糟糕。所以采用三层指标,主指标尽量用连续量。

主榜 = T3 绝对分 = 0.7·擦除率 + 0.3·力质量分 （只排 T3;分数越高名次越前）
T1 / T2 = 达标即满分 （成功率 ≥ 主办方实测标定阈值即达标显绿勾;未达标按比例 min(成功率/阈值, 1)。Δ 退出主榜,仅作诊断）
① 任务层

擦除率 / SR

任务做没做成。T1 用它做通关判定;T2/T3 中优先用连续完成度(50 集二值 SR 噪声约 ±10pp,连续指标统计效力高得多)。

② 力行为层

力质量分

做成的方式对不对——恒压贴合还是暴力硬怼。接触力在专家分布目标带(P10–P90)内的时间占比、超压占比、峰值惩罚、力方差。只由主办方 scorer 从落盘力轨迹重算。

③ 难度条件

桌高 ±3cm + 遮挡

T3 在桌高 ±3cm 随机偏移 + 视觉遮挡下计分。偏移由相机难以直读,纯视觉策略读不到桌高变化,会用力的策略才能稳擦——这正是把擦除率拉开的难度来源。力觉增益 Δ 仍由 scorer 计算留作诊断,但不进主榜分。

05 / 提交方式

自动评测,选手自报数字无效

fork 官方题目库 → 训练 → 提交 checkpoint → 评测集群自动跑分上榜。本地用同一套 harness 与 dev 种子,跑分与 dev 榜一致。

Fork 官方 Starter

脚手架、提交模板、本地评测命令内置;以锁定 commit 为 base。

训练并核验

从专家轨迹 bag 自建管线训练(可选用参考脚手架);保留可在线核验的 wandb run。

提交 checkpoint

CLI scripts/submit.py 或评测站网页上传入口(自动算 SHA256 回填);每日 ≤2 次。

ZMQ 隔离评测

选手 policy 进程(GPU)与主办方仿真 harness(CPU、私有种子)进程隔离,选手代码碰不到仿真内部状态。T3 用你 fork 的代码起服务,自定义模块全生效。

自动算分上榜

统一 score_force.py 算三层指标,逐 episode 日志落盘;Dev 榜实时,Final 榜截止冻结。

# 命令行提交(权重直接上传,后端自动算 SHA256;每队一次提交) python scripts/submit.py --server http://<评测服务器>:8080 --token <队伍令牌> \ --ckpt-file ./force.pt \ --git-url <你的fork> --git-tag <tag>

详见 学生手册 与 T3 改进点对照表。
